# 09. Расширение бота: все услуги, материалы, B2C / B2B

Этот документ описывает, как из текущего каркаса (4 услуги + рефералы) делается
полноценное меню под фактический бизнес: **8 услуг**, **5 типов материалов** и
**два режима** — частное лицо и бизнес.

Изменения в БД: расширяем enum `service_kind` (см. миграцию 6).

### Подписки (`subscription`) — отдельная сущность

Подписка — это договор «на сезон», по которому мастер регулярно приезжает (покос
+ уборка + полив и т.п.). Хранится в новой таблице `subscriptions`. Каждый визит
порождает обычный `lead` с `metadata.subscription_id`.

Пакеты (на старте 3, цена и состав — конфигурируется в Studio):

- **Базовый**: покос раз в 2 недели + уборка веток. Цена «от 8 000 ₽/мес».
- **Комфорт**: покос еженедельно + сезонная скарификация/аэрация + уборка. «от 14 000 ₽/мес».
- **Премиум**: всё из «Комфорт» + бассейн (чистка/обслуживание) + спил/корчевание по запросу. «от 25 000 ₽/мес».

## 2. Материалы

Категория «Материалы» добавляется в меню параллельно «Услугам». Обычно
интересует B2B и тех B2C, кто строится. На старте — 5 типов:

| `material_kind`   | Название           | Marки/фракции                                      | Ед. измерения | Особенности доставки         |
|-------------------|--------------------|----------------------------------------------------|---------------|------------------------------|
| `concrete`        | Бетон              | M100, M150, M200, M250, M300, M350, M400           | м³            | нужен миксер; иногда насос/манипулятор |
| `crushed_stone`   | Щебень             | 5–20, 20–40, 40–70, гранитный/гравийный/известковый | тонна / м³    | самосвал; объём = вес × плотность |
| `sand`            | Песок              | карьерный, речной, мытый, строительный             | тонна / м³    | самосвал                     |
| `cement`          | Цемент             | M400, M500; в мешках 25 / 50 кг или россыпью       | мешок / тонна | паллета; манипулятор для разгрузки |
| `brick`           | Кирпич             | рядовой, облицовочный, силикатный, керамический    | шт. / поддон  | манипулятор обязателен        |

### Структура

- `materials` — тип материала (`concrete`, `crushed_stone`, …).
- `material_grades` — конкретные марки/фракции у каждого типа: `M200`,
  `гранитный 5–20`, `карьерный` и т.д., с примерной ценой.
- `material_orders` — отдельная таблица заказов (схожа с `leads`, но:
  `material_id`, `grade_id`, `quantity`, `unit`, `delivery_address`,
  `needs_pump` boolean, `needs_manipulator` boolean, `desired_date`).

Бонусы и скидки лояльности применяются к `material_orders` так же, как к
`leads` — RPC `compute_discount_for_contact` принимает дополнительный
параметр-категорию (или у нас будет отдельная функция
`compute_material_discount`).

### Воронка для бетона (пример B2B)

```
1. material            (выбрана из меню)
2. concrete:grade      [M150] [M200] [M250] [M300] [M350] [M400] [не знаю]
3. quantity            [1–3 м³] [4–6] [7–10] [10+] [✏️ указать]
4. when                [сегодня] [завтра] [эти выходные] [на этой неделе] [✏️ дата]
5. extras              ☐ нужен бетононасос  ☐ нужен манипулятор  ☐ только разгрузка с борта
6. delivery_address    [✏️ адрес]
7. confirm             карточка-резюме + цена за куб × объём + скидка
8. phone               (если нет в контактах)
9. done                "✅ Заявка принята, менеджер позвонит подтвердить и оплатить"
```

### Воронка для щебня

То же, но шаги: `kind` (фракция + порода) → `quantity` (тонн или м³) →
`when` → `delivery_address` → `confirm` → `phone`.

## 3. B2C vs B2B — разделение

### Где спрашиваем

При **первом** `/start` после выбора языка:

```
👋 Здравствуйте! Чтобы я предложил подходящее меню — подскажите:
вы оформляете заказ для частного дома или для компании / стройки?
```

```
[ 🏡 Для частного дома ]   [ 🏗 Для компании / стройки ]
```

`callback_data`: `ctype:b2c`, `ctype:b2b`. Сохраняем в `contacts.customer_type`. При следующем `/start` сразу показываем главное меню.

### Меню B2C (физическое лицо)

```
[ 🌱 Покос ]            [ 🌿 Скарификация ]
[ 🪓 Расчистка ]        [ 🏊 Бассейн ]
[ 🪚 Спил и пни ]       [ 🔥 Сварка ]
[ 🚜 Вспашка ]          [ 📅 Абонентка ]
─────────────────────────────────────────
[ 🧱 Стройматериалы ]   [ 📋 Мои заказы ]
[ 🎁 Пригласить друга ] [ ☎️ Оператор ]
```

12 кнопок (3 строки по 2 + ещё 2 строки сервисных). На Telegram это
помещается в один экран.

### Меню B2B (бизнес/стройка)

```
[ 🧱 Материалы ]        [ 🛠 Услуги ]
─────────────────────────────────────────
[ 📋 Мои заказы ]       [ 🎁 Пригласить ]
[ 🤝 Договор / счёт ]   [ ☎️ Менеджер ]
```

Там 2 верхних кнопки ведут в подменю:

- `nav:materials` →

  ```
  [ 🪨 Щебень ] [ 🌾 Песок ]
  [ 🏗 Бетон ]  [ 🧊 Цемент ]
  [ 🧱 Кирпич ]
  [ ◀️ Назад ]
  ```

- `nav:services` (B2B) — те же 8 услуг, но с пометкой «оформление по
  договору» и кнопкой «Запросить КП» вместо «Подтвердить».

### Где B2C и B2B расходятся в логике

- **Цены.** `materials.price_*` базовые. Для B2B оставляем «оптовую» цену
  такую же на старте, но в `metadata` подписи «цены на B2B уточняет менеджер».
- **Подтверждение.** B2C сразу видит «вилку» и подтверждает — лид сразу `new`.
  B2B видит ту же вилку, но финальная цена «по запросу» — лид создаётся в
  `quoted` после согласования с менеджером (через n8n).
- **Реферал.** Та же программа, но в B2B-боте текст «Приведите подрядчика —
  500 ₽ скидки на первый заказ материалов».
- **Лояльность.** Те же 5 % / 10 %, считаются от **общего** числа `done`
  (услуги + материалы) в году.
- **Документы.** В B2B-меню кнопка «🤝 Договор / счёт» → запрос реквизитов
  (ИНН, КПП, юрадрес) и отправка их менеджеру через n8n.

### Что в БД

- `contacts.customer_type text default null` — `'b2c' | 'b2b'`. Если null —
  на старте предлагаем выбрать.
- `contacts.company_inn text`, `company_name text`, `company_kpp text` —
  для B2B-реквизитов (заполняются по запросу).
- `material_orders.customer_type` — денормализованно для отчётов.

## 4. Изменения в коде

### Миграция `20260507000006_services_materials_b2b.sql`

См. файл. Содержимое:

1. Расширение `service_kind` (`alter type … add value`).
2. Поля в `contacts`: `customer_type`, `company_*`.
3. Новые таблицы: `materials`, `material_grades`, `material_orders`,
   `subscriptions`, `subscription_visits`.
4. RPC: `set_customer_type`, `create_material_order`,
   `compute_discount_unified` (переписана версия `compute_discount_for_contact`,
   которая считает по `done` обоих типов), триггер
   `bump_total_orders` для `material_orders`.
5. View `v_my_all_orders` — UNION `leads` и `material_orders` для одного
   списка «Мои заказы».
6. RLS на новые таблицы. Revoke от anon, grant service_role.
7. Все RPC с `set search_path = public, pg_temp`.

### `lib/funnels.ts` — расширение

- `ServiceKind` пополняется новыми значениями: `welding`, `tilling`,
  `weed_removal`, `pool_maintenance`, `subscription`.
- `MaterialKind = 'concrete' | 'crushed_stone' | 'sand' | 'cement' | 'brick'`.
- Новые лейблы и шаблоны цен.
- Новый `Screen`: `material_order`, `pick_customer_type`, `b2b_documents`.
- `OrderStep` для материалов:
  `kind → grade → quantity → when → extras → delivery_address → confirm → phone → done`.
- Расширенный `SessionState` с `customerType`, `materialKind`, `gradeCode`,
  `quantity`, `unit`, `needsPump`, `needsManipulator`, `deliveryAddress`.

### `lib/orders.ts` — добавки

- `createMaterialOrder()`, `listMyOrdersUnified()` (берёт из view
  `v_my_all_orders`), `setCustomerType()`, `getCustomerType()`.

### Клавиатуры

`lib/telegram.ts` и `lib/max.ts`:

- `customerTypeKeyboard()` — выбор B2C / B2B.
- `mainMenuB2cKeyboard()` — 12 кнопок частнику.
- `mainMenuB2bKeyboard()` — 6 кнопок бизнесу.
- `materialsMenuKeyboard()` — 5 материалов + назад.
- `gradeKeyboard(materialKind)` — марки/фракции для конкретного материала.
- `quantityKeyboard(materialKind)` — бакеты объёма.
- `extrasKeyboard()` — multi-select: насос / манипулятор / только разгрузка.

### Webhooks

В `onMessage` после `upsertContactByIdentity` добавляется проверка:
если `customer_type` пустой — показать `pick_customer_type`. После выбора —
ветви `mainMenuB2cKeyboard` / `mainMenuB2bKeyboard`.

В `onCallback` появляется `scope='mat'`:

```
mat:concrete | mat:crushed_stone | mat:sand | mat:cement | mat:brick
grade:<material>:<code>
qty:<material>:<bucket>
extra:pump | extra:manipulator | extra:offloading
```

Воронка материалов — отдельная функция `advanceMaterialOrder()`, аналогичная
существующему `advanceFunnel()` для услуг.
